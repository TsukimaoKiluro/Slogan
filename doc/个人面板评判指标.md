# 个人学习面板评判指标

> 本系统面向大学生群体，用于全面评估和展示个人学习状况。
> 
> **技术栈**：Vue 3 + ECharts + Express + SQLite

## 指标体系概览

| 指标模块 | 图表类型 | 目的 | 数据表 |
|---------|---------|------|--------|
| 知识基础 | 雷达图 | 展示多学科/知识模块的掌握程度 | `knowledge_base` |
| 认知风格 | 条形图 | 分析学习方式偏好 | `cognitive_style` |
| 易错点偏好 | 热力图 | 识别薄弱知识点 | `error_points` |
| 学习目标 | 圆环图 | 追踪目标完成进度 | `study_goals` |
| 兴趣方向 | 词云 | 展示学习兴趣分布 | `interests` |
| 学习节奏/效率 | 折线图 | 分析时间序列学习表现 | `study_records` |

## 数据库设计

### 表结构

#### 1. knowledge_base - 知识基础表

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键 |
| user_id | TEXT | 用户ID，默认 'default' |
| subject | TEXT | 学科标识（关联 subjects.name） |
| score | INTEGER | 掌握程度 (0-100) |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

> **学科来源**：知识基础表中的 `subject` 字段关联 `subjects` 表的 `name`，显示名称从 `subjects.display_name` 获取。

> **默认学科**：`math`(高数), `linear_algebra`(线代), `probability`(概率论), `physics`(大物), `major`(专业课), `english`(英语), `programming`(编程), `politics`(政治)

#### 2. subjects - 学科表（v1.0.1 新增）

| 字段 | 类型 | 说明 |
|-----|------|------|
| name | TEXT | 学科标识（主键） |
| display_name | TEXT | 显示名称 |
| icon | TEXT | 图标标识 |
| color | TEXT | 主题颜色 |
| sort_order | INTEGER | 排序权重 |

> **用途**：动态管理学科列表，knowledge_base 等表的学科信息来源于此表。

#### 2. cognitive_style - 认知风格表

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键 |
| user_id | TEXT | 用户ID |
| style | TEXT | 风格标识 |
| score | INTEGER | 偏好程度 (0-100) |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

**风格标识**：`hands_on`(动手操作), `visual`(视觉学习), `auditory`(听觉学习), `summarize`(归纳总结), `social`(社交学习)

#### 3. error_points - 易错点表

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键 |
| user_id | TEXT | 用户ID |
| topic | TEXT | 知识点标识 |
| question_type | TEXT | 题型标识 |
| error_count | INTEGER | 错误次数 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

**知识点标识**：`function`(函数), `equation`(方程), `geometry`(几何), `probability`(概率), `algorithm`(算法), `proof`(证明)

**题型标识**：`choice`(选择题), `fill`(填空题), `essay`(解答题)

#### 4. study_goals - 学习目标表

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键 |
| user_id | TEXT | 用户ID |
| title | TEXT | 目标标题 |
| description | TEXT | 目标描述 |
| deadline | TEXT | 截止日期 |
| status | TEXT | 状态: `completed`(已完成), `in_progress`(进行中), `pending`(未开始) |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

#### 5. interests - 兴趣方向表

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键 |
| user_id | TEXT | 用户ID |
| interest | TEXT | 兴趣标识 |
| weight | INTEGER | 权重/热度 (0-100) |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

**兴趣标识**：`programming`(编程), `math`(数学), `physics`(物理), `music`(音乐), `reading`(阅读), `science`(科学), `art`(绘画), `history`(历史), `biology`(生物), `game_design`(游戏设计), `ai`(AI), `astronomy`(天文)

#### 6. study_records - 学习记录表

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | INTEGER | 主键 |
| user_id | TEXT | 用户ID |
| record_date | TEXT | 记录日期 (YYYY-MM-DD) |
| efficiency | INTEGER | 学习效率 (0-100) |
| duration | REAL | 学习时长 (小时) |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

## API 接口文档

**Base URL**: `http://localhost:3001/api`

### 个人面板 API

#### 初始化面板数据
```
POST /api/profile/init
```
初始化用户的默认面板数据结构。

---

#### 获取所有面板数据
```
GET /api/profile/data
```
返回所有面板指标的聚合数据。

**响应示例**:
```json
{
  "success": true,
  "data": {
    "knowledge": [...],
    "cognitiveStyle": [...],
    "errorPoints": [...],
    "goals": [...],
    "goalStats": { "completed": 0, "in_progress": 0, "pending": 0, "total": 0 },
    "interests": [...],
    "studyRecords": [...]
  }
}
```

---

| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | `/api/profile/knowledge` | 获取知识基础数据 |

**PUT 请求体**:
```json
{
  "scores": [
    { "subject": "math", "score": 85 },
```

## 数据来源与计算

### 1. 知识基础
- **数据来源**: 考试成绩、自测评估、作业完成质量
- **计算方式**: 各学科得分 (0-100)
- **更新频率**: 按需手动更新或考试后自动更新

### 2. 认知风格
- **数据来源**: 学习风格问卷测试、学习行为分析
- **计算方式**: 各风格偏好程度 (0-100)
- **更新频率**: 初始评估后定期更新

### 3. 易错点偏好
- **数据来源**: 错题记录、测试数据分析、作业错误统计
- **计算方式**: 错误次数累计
- **更新频率**: 每次练习/测试后自动记录

### 4. 学习目标
- **数据来源**: 用户自设定目标
- **计算方式**: 状态计数 (已完成/进行中/未开始)
- **更新频率**: 用户手动管理

### 5. 兴趣方向
- **数据来源**: 学习历史记录、课程选择偏好、内容浏览行为
- **计算方式**: 基于行为的权重计算 (0-100)
- **更新频率**: 根据学习活动动态更新

### 6. 学习节奏/效率
- **数据来源**: 学习日志、番茄钟记录、任务完成统计
- **计算方式**: 
  - 学习效率: 完成任务量/标准任务量 × 100
  - 学习时长: 实际学习时间累计
- **更新频率**: 每日记录

## 设计原则

1. **数据驱动**: 各指标数据应来自实际学习行为，而非手动输入
2. **实时更新**: 面板数据应随学习活动动态更新
3. **可视化优先**: 选择最能传达信息的图表类型
4. **用户友好**: 布局清晰，交互友好
5. **可扩展性**: 支持多用户和数据迁移

## 快速开始

1. **启动后端服务**:
```bash
npm run server
```

2. **访问个人面板**:
在浏览器打开 `http://localhost:3000/profile`

3. **API 测试**:
```bash
# 初始化数据
curl -X POST http://localhost:3001/api/profile/init

# 获取所有数据
curl http://localhost:3001/api/profile/data
```
